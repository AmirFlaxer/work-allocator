# אזהרה לפני ייצוא/פרסום עם משבצות ריקות - עיצוב (בקלוג #9)

**תאריך:** 2026-07-18
**סטטוס:** מאושר ע"י אמיר (כולל עדכון מדריך המשתמש)

## מהות

לפני שלוש הפעולות "כלפי חוץ" - פרסום לצוות, ייצוא PNG, ייצוא Excel - אם יש בשיבוץ
הנוכחי משבצות ריקות, מוצג דיאלוג אישור: "יש X משבצות ריקות בשיבוץ - להמשיך?".
"המשך בכל זאת" מריץ את הפעולה; "ביטול" סוגר. כשאין משבצות ריקות - אפס שינוי
בהתנהגות (אין חיכוך במקרה הרגיל).

## מימוש (הכל ב-`src/pages/Index.tsx`)

- **קיים כבר:** `emptySlots` (useMemo, שורה ~891) סופר משבצות ריקות בשיבוץ הנוכחי.
- **state חדש:** `pendingExportAction: "publish" | "png" | "excel" | null`.
- **שומר:** פונקציה `guardEmptySlots(action)` - אם `schedule` קיים ו-`emptySlots > 0`,
  שומרת את הפעולה ב-state ופותחת דיאלוג; אחרת מריצה ישר את ה-handler המקורי
  (`handlePublish` / `handleExportToImage` / `handleExportToExcel`).
- שלושת הכפתורים הקיימים (שורות ~1199, ~1286, ~1289) קוראים לשומר במקום ישירות.
- **AlertDialog משותף אחד** בדפוס הקיים של אישור-מחיקת-עמדה (שורה ~1450):
  - כותרת: `יש {emptySlots} משבצות ריקות בשיבוץ`
  - תיאור לפי הפעולה: פרסום - "השיבוץ יפורסם לצוות עם המשבצות הריקות. להמשיך?";
    ייצוא (png/excel) - "הקובץ ייווצא עם המשבצות הריקות. להמשיך?"
  - כפתורים: "המשך בכל זאת" (מריץ את הפעולה השמורה וסוגר) / "ביטול" (רק סוגר).
- אין לוגיקה טהורה חדשה - אין בדיקות vitest חדשות. שערים: tsc + vitest קיימים +
  lint (בלי אזהרות חדשות) + build + אימות ויזואלי חי.

- אזהרות בדוח החודשי (MonthlyReport) - הבקלוג מדבר על השיבוץ השבועי בלבד.
- "אל תציג שוב" / העדפת השתקה.
